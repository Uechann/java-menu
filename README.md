# 프로젝트 요구사항
한 주의 점심 메뉴 추천해주는 서비스이다.
코치들은 월, 화, 수, 목, 금요일에 점심식사를 한다.

1. 월요일에 추천할 카테고리는 무작위이다.
2. 각 코치가 월요일에 먹을 메뉴를 추천한다.
3. 화, 수, 목, 금요일에 대해 1, 2, 과정을 반복한다.

- 코치 이름은 최소 2글자, 최대 4글자
- 코치는 최소 2명, 최대 5명까지 식사를 함께 한다.
- 각 코치는 최소 0개, 최대 2개의 못 먹는 메뉴가 있다. (,로 구분하여 입력)
- 한 주에 같은 카테고리를 최대 2회까지만 고를 수 있다.
- 각 코치에게 한 주에 중복되지 않는 메뉴를 추천해야한다. 
  - 예시) 
    - 구구: 비빔밥, 김치찌개, 쌈밥, 규동, 우동 -> 한식 3회라서 불가능
    - 토미: 비빔밥, 비빔밥, 규동, 우동, 볶음면 -> 같은 메뉴 존재 불가능
    - 제임스: 비빔밥, 김치찌개, 스시, 가츠동, 짜장면 -> 매일 다른 메뉴여서 가능
    - 포코: 비빔밥, 김치찌게, 스시, 가츠동, 짜장명 -> 제임스와 같은 메뉴지만, 가능

# 기능 목록

## 코치 이름 입력 기능
예외 상황
- 이름 최소 2글자, 최대 4글자
- 코치는 최소 2명, 최대 5명까지 함께 식사

## 한 코치의 못 먹는 메뉴 입력 기능
예외 상황
- 못 먹는 음식 최소 0개, 최대 2개까지 가능 

## 메뉴 추천 기능
세부 기능 
- 카테고리: 일식, 한식, 중식, 아시안, 양식 (5개)
- 카테고리 별 메뉴: README.md 파일 참고
- 월화수목금 12345 요일별 같은 카테고리 최대 2회
- 한 코치에게 중복된 메뉴를 추천 안된다.
